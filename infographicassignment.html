The Lockeed-Martin Kill-Chain

R E C O N
Open Source Intelligence or OSINT is the process of gathering information on a potential target for an attack. This information gathered can consist of anything from social media to home address and email. This part of the killchain is difficult to detect and mitigate because it is technically legal.




W E A P O N I Z E
Weaponization is the act of taking informaion gathered about a target and crafting or selecting an exploit and delivery method. This can look like malware development, cve research, phishing page creation, phishing email writing, or getting stolen creds




D E L I V E R Y
Delivery is the action of sending the weaponized attack method to the target. This can look like phishing emials, malvertising, code injection, or use of ingress methods to transfer the payload to the target computers.

Preventitive Meauseres: firewall rules, cloud scanning, user vigilance





E X P L O I T
Exploitaition is the execution of the weaponized code. This can look like an executable progran being run on a computer, malicous code exectution by a server via unsanitiezed user input, or credential theft using a phishing page.

Preventative measures: antivirus software, privlage management





I N S T A L L
Install refers to the installation of a backdoor, or more code to execute to perform actions or spawn a shell to interact with the target device.

Preventative measures: firewall rules





C 2
C2 is an abbreviation for Command And Control. This refers to the device that connects back to the victim on the threat actor's behalf.




A C T I O N S
Actions refer to the actions taken by the attakcer after the payload has been executed and the C2 has been set up. This can be data exfiltration or corruption or an array of other options.
